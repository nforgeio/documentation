# Deploy A NeonKUBE Cluster on Azure

## Prerquisites

Make sure you have neon-desktop installed. To install, follow the neon-desktop
installation instructions [here](./neon-desktop)

## Creating a cluster

### Azure CLI

Install the
[Azure CLI](https://api-docs.neonforge.com/neonkube/api/Neon.Kube.ClusterDef.ClusterDefinition.html).

### Azure RBAC

Run the following command:

```bash
az ad sp create-for-rbac --name USER --role owner --scopes /subscriptions/AZURE_SUBSCRIPTION_ID
```

The output will be something like:

```json
{
  "appId": "AZURE_CLIENT_ID",
  "displayName": "example-app-name",
  "name": "http://example-app-name",
  "password": "AZURE_CLIENT_SECRET",
  "tenant": "AZURE_TENANT_ID"
}
```
### Accept offer terms

:::info Info

Replace `neonforge:neonkube:0-10-0-beta-5:0.10.0` with the version you want to use.

For example version 1.0.0 would be `neonforge:neonkube:1-0-0:1.0.0`

:::

View offer terms:
```bash
az vm image terms show --urn neonforge:neonkube:0-10-0-beta-5:0.10.0
```

Accept offer terms:
```bash
az vm image terms accept --urn neonforge:neonkube:0-10-0-beta-5:0.10.0
```


### Cluster Definition

Create a
[Cluster Definition](https://api-docs.neonforge.com/neonkube/api/Neon.Kube.ClusterDef.ClusterDefinition.html)
that describes the cluster.

:::caution Warning

Make sure that the Azure account has enough
[Resource Quota](https://learn.microsoft.com/en-us/azure/azure-resource-manager/management/azure-subscription-service-limits)
to deploy the cluster.

:::

Be sure to substitute the following from the previous steps:

- `AZURE_SUBSCRIPTION_ID`
- `AZURE_TENANT_ID`
- `AZURE_CLIENT_ID`
- `AZURE_CLIENT_SECRET`

```yaml
name: azure-tiny
datacenter: westus2
purpose: development
timeSources:
  - pool.ntp.org
kubernetes:
  allowPodsOnControlPlane: true
storage:
hosting:
  environment: azure
  cloud:
    prefixResourceNames: default
  azure:
    subscriptionId: AZURE_SUBSCRIPTION_ID
    tenantId: AZURE_TENANT_ID
    clientId: AZURE_CLIENT_ID
    clientSecret: AZURE_CLIENT_SECRET
    region: westus2
    disableProximityPlacement: false
    defaultVmSize: Standard_E4vs_v5
nodes:
  control-0:
    role: control-plane
```

Save the Cluster Definition.

### Creating the Cluster

1. Right click the neon-desktop icon in the taskbar > Deploy cluster
   <Image img={require("./assets/deploy-cluster.png")} style={{ width: 300 }} />
2. When the dialog pops up, select the Cluster Definition file you saved
   earlier.
3. Grab a coffee and wait for neon-desktop to create the cluster.
